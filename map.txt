1.  PROJECT OVERVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Cognitive AI is a local-first desktop application that:
  • Indexes documents (PDF, Markdown, TXT) from folders
  • Provides dual-mode search (rule-based + AI-assisted)
  • Answers questions via RAG using local Ollama models
  • Remembers conversation history across sessions
  • Runs entirely offline — zero token cost, full data privacy

  A. DOCUMENT IMPORT (app.py → core.py)
  ─────────────────────────────────────────────────────────────

  User clicks "Import Folder" or "+ Add File"
       │
       ▼
  add_folder() or add_file()
       │
       ├── File selected → _show_folder_import_dialog()
       │   ┌──────────────────────────────────────────┐
       │   │ Dialog shows all detected files           │
       │   │ with checkboxes (default all selected).   │
       │   │ "Select All" / "Deselect All" toggle.     │
       │   │ Already-imported files grayed out.        │
       │   └──────────────────────────────────────────┘
       │       │
       │       ▼
       │   _process_selected_files(files)
       │
       ▼
  DocumentProcessor.process_file(file_path)
       │
       ├── PDF  → extract_text_from_pdf()   → PdfReader per page
       ├── MD   → extract_text_from_markdown() → read entire file
       └── TXT  → extract_text_from_markdown() → read entire file
       │
       ▼
  DocumentProcessor.chunk_text(text, page, filename)
       │
       ├── Splits text into chunks of chunk_size (default: 500)
       ├── chunk_overlap between chunks (default: 50)
       └── Each chunk gets: {chunk, pdf_name, page, start_pos, end_pos}
       │
       ▼
  VectorStore.add_chunks(chunks, metadata)
       │
       ├── For each chunk: get_embedding() via Ollama/all-minilm:l6-v2
       ├── normalize_embedding() (L2 norm → unit vector)
       ├── index.add(embeddings_np) → FAISS IndexFlatIP
       └── metadata saved to JSON file
       │
       ▼
  Database.add_document(filename, filepath, filetype, chunk_count)
  Database.update_system_stats(total_chunks)
       │
       ▼
  UI updates: refresh_document_list(), update_stats(),


  B. SIDEBAR SEARCH (search_engine.py)
  ─────────────────────────────────────────────────────────────

  User types keyword + presses Enter or clicks 🔍
       │
       ▼
  _run_search()
       │
       ├── mode = "Quick (no AI)" or "Smart (+AI summary)"
       │
       ▼
  DualSearchEngine.search(keyword, file_paths, mode)
       │
       ├── PHASE 1 (always, 0% AI):
       │   RuleBasedSearcher.search_files(paths, keyword)
       │   │
       │   ├── For each file: extract_text_from_file()
       │   ├── re.finditer(keyword, text, IGNORECASE)
       │   ├── Build excerpt with [keyword] markers
       │   ├── Context window: 300 chars before/after each match
       │   └── Rank by match frequency × relevance multiplier
       │   │
       │   └── Returns: [ {file_path, filename, page, excerpt, 
       │                   full_context, score}, ... ]
       │
       ├── PHASE 2 (only if mode="smart", ~20% AI):
       │   LightweightSummarizer.batch_summarize(top_5, keyword)
       │   │
       │   ├── Only top 5 pre-located segments get AI
       │   ├── Model: qwen3:0.6b (low tier) or qwen2.5:3b (high)
       │   ├── temperature=0.1, max_tokens=256, top_p=0.5
       │   ├── Prompt: "Extract only info relevant to question"
       │   └── Returns: segments with ai_summary field added
       │
       └── Returns: {mode, keyword, total_matches, results, 
                      timing, ai_usage_pct}
       │
       ▼
  _display_search_results()
       │
       ├── Results shown in sidebar listbox
       ├── Each result: "filename (p.N)" + ai_summary (if smart)
       └── Status bar: "N matches | search: X.Xs | AI: Y.Ys | AI: Z%"
       │
       ▼
  Double-click result → _open_search_result()
       │
       └── Opens document in viewer panel


  C. BROWSE STORAGE (Cross-Location Search)
  ─────────────────────────────────────────────────────────────

  User clicks "Browse Storage" → selects a folder
       │
       ▼
  _do_browse(folder)
       │
       ├── If keyword entered:
       │   CrossLocationFinder.find_files_by_content(keyword, roots)
       │   │  ├── FileScanner.scan_multiple_roots() → all files
       │   │  └── RuleBasedSearcher.search_files() → find matches
       │   │
       │   └── Results: [ {path, filename, folder, 
       │                   match_count, excerpts}, ... ]
       │
       └── If no keyword:
           FileScanner.scan_folder(folder) → file manifest
           └── Results: [ {path, filename, ext, size_kb, modified}, ... ]


  D. CHAT / RAG QUERY (app.py → smolagent_helper.py)
  ─────────────────────────────────────────────────────────────

  User types question + Enter or clicks "Send →"
       │
       ▼
  send_message()
       │
       ├── If active_document connected:
       │   query = "[Context: doc_name] user_question"
       │
       ▼
  _process_message(query)
       │
       ├── 1. KnowledgeAgent.search_knowledge(query)
       │   │   └── VectorStore.search(query_embedding, top_k=3)
       │   │       → [ {chunk, pdf_name, page, score}, ... ]
       │   │
       ├── 2. KnowledgeAgent.format_context(results)
       │   │   → "--- Source 1 (Page 2, report.pdf) ---\n{chunk}..."
       │   │
       ├── 3. KnowledgeAgent.build_prompt(query, context)
       │   │   → "Use ONLY provided context. If no answer, say so."
       │   │
       ├── 4. KnowledgeAgent.calculate_confidence(results)
       │   │   → confidence% from top score × 100
       │   │
       ├── 5. ollama_client.generate(model=qwen2.5:3b, prompt, 
       │   │        options={temp:0.7, max_tokens:1024, top_p:0.9})
       │   │   → ai_response
       │   │
       ├── 6. Database.add_memory(query, ai_response)
       │
       ▼
  _update_chat_response(message, confidence, latency, sources)
       │
       └── append_message("AI", message, confidence, latency, sources)
           └── Shows: confidence% badge, retrieval latency, 
                       source list (filename, page, score)

       ▼                  ▼
  Conversation Memory   FAISS Search

  G. EVALUATION PIPELINE (eval.py)
  ─────────────────────────────────────────────────────────────

  python eval.py or double-click run_eval.bat
       │
       ▼
  run_evaluation()
       │
       ├── Loads Config
       ├── Processes sample_report.pdf if not already indexed
       ├── Loads evals/dataset.json (20 test cases)
       │
       ├── For each case:
       │   ├── VectorStore.search(case.question)
       │   ├── Build prompt → ollama.generate()
       │   ├── score_response() → keyword matching
       │   └── Record: score, passed, timing, category, type
       │
       └── Computes: pass_rate, avg_score, category/type/difficulty scores
           └── Saves to evals/results.json

           │                                   │
           ▼                                   ▼
  ┌──────────────────┐              ┌──────────────────────┐
  │ MODE: QUICK      │              │ MODE: SMART          │
  │ (0% AI)          │              │ (~20% AI)            │

  │                  │              │ → LightweightSum-    │
  │ Latency: ~0.01s  │              │   marizer (AI)       │
  │ AI Usage: 0%     │              │ → summarized answers │
  │                  │              │                      │
  │ Best for: quick  │              │ Latency: ~1-3s       │
  │ keyword lookup   │              │ AI Usage: ~20%       │
  │                  │              │                      │
  │                  │              │ Best for: complex    │
  │                  │              │ questions needing    │

5.  CONFIGURATION REFERENCE (config.yaml)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  models:
    chat: qwen2.5:3b                # High-resource chat model
    chat_low: qwen3:0.6b            # Low-resource fallback
    embedding: all-minilm:l6-v2     # Embedding model (384-dim)
    search_chat: qwen3:0.6b         # Search summarization model

  ollama:
    host: http://localhost:11434    # Ollama server

  paths:
    database: data/app.db           # SQLite file
    embeddings: data/embeddings/    # FAISS index + metadata

  chunking:
    chunk_size: 500                 # Characters per chunk
    chunk_overlap: 50               # Overlap between chunks

  vector_store:
    top_k: 3                        # Default retrieval count
    use_normalized_embeddings: true # IndexFlatIP (cosine sim)

  search:
    default_mode: quick             # quick (0% AI) or smart (~20% AI)
    context_chars: 300              # Context window for keyword matches
    max_results: 20                 # Max search results
    model_tier: low                 # low (qwen3:0.6b) or high (qwen2.5:3b)
    search_roots:                   # Cross-location search paths
      - data/pdfs
      - data/test_documents

6.  WORKING STYLE — BEST PRACTICES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ┌─ AI USAGE PRINCIPLE ────────────────────────────────────┐
  │  Rule-based search is ALWAYS the primary path.          │
  │  AI is summoned ONLY for summarization of pre-located   │
  │  text segments. Target: ≤20% AI processing per query.   │
  │                                                          │
  │  Quick mode  = 0% AI, instant results                   │
  │  Smart mode  = ~20% AI, adds synthesis to top results   │
  └──────────────────────────────────────────────────────────┘

  ┌─ PERFORMANCE TIPS ──────────────────────────────────────┐
  │  • For basic laptops: set model_tier: low               │
  │  • For keyword lookups: use Quick mode (instant)        │
  │  • For understanding: use Smart mode (adds AI summary)  │
  │  • Large folders: import once, then search is fast      │
  │  • FAISS index persists across restarts                 │
  │  • Conversation memory persists in SQLite               │
  └──────────────────────────────────────────────────────────┘

  ┌─ UI NAVIGATION ─────────────────────────────────────────┐

  │  Status bar (bottom): System health + project stats     │
  │  Header (top): Import, Reset, Viewer toggle             │

7.  COMPLETE WORKFLOW SCENARIOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  SCENARIO A: "Find the author name in Q3_report.pdf"
  ─────────────────────────────────────────────────────────────
  1. Import folder containing Q3_report.pdf
  2. Single-click report in sidebar → connects to chat
  3. Type "Who is the author?" in chat → Send
  4. System prepends [Context: Q3_report.pdf] to query
  5. VectorStore filters search to only that document
  6. Ollama returns answer with source citation

  SCENARIO B: "Search all documents for 'budget'"
  ─────────────────────────────────────────────────────────────
  1. Type "budget" in sidebar search bar
  2. Set mode to "Quick (no AI)" for instant results
  3. System: RuleBasedSearcher scans all indexed files
  4. Results show: "report.pdf (p.5)" with matching excerpt
  5. Double-click result → opens document at that page

  SCENARIO C: "Summarize what documents say about AI"
  ─────────────────────────────────────────────────────────────
  1. Type "AI" in sidebar search bar
  2. Set mode to "Smart (+AI summary)"
  3. Phase 1: RuleBasedSearcher finds all matches
  4. Phase 2: LightweightSummarizer summarizes top 5 segments
  5. Results show: "report.pdf (p.3) ↳ AI is transforming..."

  SCENARIO D: "Find my to-do list project folder"
  ─────────────────────────────────────────────────────────────
  1. Click "Browse Storage"
  2. Select root folder (e.g., Documents)
  3. Type "to-do" in search bar
  4. CrossLocationFinder.find_files_by_content("to-do", roots)
  5. Scans all supported files in Documents and subfolders
  6. Returns files containing "to-do" with match counts

  SCENARIO E: "Import only specific files from a folder"
  ─────────────────────────────────────────────────────────────
  1. Click "Import Folder" → select folder
  2. Dialog shows all 20 detected files, all checked
  3. Uncheck the ones you don't want
  4. Click "Import Selected"
  5. Only checked files get processed and indexed
