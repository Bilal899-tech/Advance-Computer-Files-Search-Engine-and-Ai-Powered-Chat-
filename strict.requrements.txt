### 📁 Required Project Files & Assets

* **`README.md` & Design Doc (500–1500 words)**
* **What it does:** Explain that it parses uploaded PDFs, extracts structure (headings/searchable text), uses a tool call to pull direction/tracking info, and uses a local model (`Qwen:0.6B`) to summarize the notes in conversational language.

* **`evals/dataset.json` (The 15% Weight Backbone)**
* Create **20 distinct test cases** before you finalize the code.
* Include 10 factual questions directly from your PDFs (e.g., *"What is the policy on X?"*) and 10 tracking scenario queries designed to force your model to call your tracking tool.


* **`evals/results.json` (Evidence of Iteration)**
* Show a baseline run where the model might miss a tool call or truncate a summary, and an improved run after you tweak your prompt structure.

* Hardcode the tracking function tool clearly so the model can easily invoke it via a JSON print or a structured string block.


* **Proof of Sovereign Compute Asset**
* Take a quick screenshot of your SPUR environment or include a log capturing an `nvidia-smi` run showing the local `Qwen` model loaded into GPU VRAM.
